The 6 Pillars of Data Quality

To measure quality, we must first define it. These six dimensions form the standard framework (ISO 25012) for assessing data health.

1. Accuracy

Does the data correctly represent the real-world entity? (e.g., Is the customer's age actually 25?)

2. Completeness

Are all required data fields present? (e.g., No missing sensor readings in the time series.)

3. Consistency

Does data align across different systems? (e.g., CRM and Billing show the same address.)

The AI-Readiness Gap

Traditional software can handle some noise. AI systems amplifies it. An "AI-Ready" dataset requires more than just correctness; it needs representativeness and lack of bias.

Error Propagation: Traditional vs. AI Systems

Insight: In AI, a small data bias (e.g., non-representative training set) compounds into massive model failure, whereas traditional logic might just throw a single exception.

What makes data "AI-Ready"?

  • ● Representativeness: Does the training data cover all edge cases (e.g., Region D in the book example)?
  • ● Label Quality: Are the annotations consistent and unbiased?
  • ● Traceability: Can we trace a model decision back to the specific training batch?
Example Failure Mode

In the book's SalesNet example, the model failed for "Region D" because it learned a spurious correlation from a single data point. This is a Completeness failure manifesting as an Accuracy failure in prediction.

Case Study: Motor Vibration Analysis

Detecting anomalies in time-series data using FFT (Fast Fourier Transform) and Autoencoders. This visualizes the `motor_simulation-service` output.

The Scenario

A motor vibrates normally at 25Hz and 67Hz. At T=2.5s, a fault injects new frequencies (13, 45, 89Hz).

Sampling Rate: 1000Hz Noise Level: 0.65

The Solution

An Autoencoder is trained on "clean" data. When it sees the "noisy/faulty" data, the reconstruction error spikes, flagging an anomaly.
